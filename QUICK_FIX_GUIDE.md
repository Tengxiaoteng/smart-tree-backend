# 🚀 快速修复指南

## 问题

你看到错误："父节点不存在"

**原因**: 你之前创建的节点只在 localStorage 中，不在数据库里。当你创建子节点时，后端找不到父节点。

## ✅ 解决方案（3选1）

### 方案 1: 迁移现有数据（推荐）⭐

使用迁移工具将 localStorage 中的数据导入到数据库：

1. **打开迁移工具**
   ```
   在浏览器中打开：
   http://localhost:3000/migrate_to_backend.html
   ```

2. **按步骤操作**
   - 点击 "1. 检查本地数据" - 查看有多少数据
   - 点击 "2. 开始迁移" - 将数据导入数据库
   - 等待迁移完成
   - 刷新页面

3. **验证**
   - 退出登录
   - 重新登录
   - 检查节点是否还在

### 方案 2: 清空重新开始（最简单）

如果你的数据不重要，可以清空重新开始：

1. **清空浏览器数据**
   - 按 F12 打开开发者工具
   - 进入 Application → Local Storage
   - 删除 `smart-tree-storage`
   - 刷新页面

2. **重新创建**
   - 创建新的主题
   - 创建新的节点
   - 这次会自动保存到数据库

### 方案 3: 只创建根节点（临时方案）

暂时只创建根节点（没有父节点的节点）：

1. 创建主题
2. 只创建第一层节点（不要创建子节点）
3. 等所有根节点都同步到数据库后
4. 再创建子节点

## 🔍 如何验证数据已保存到数据库

### 方法 1: 退出登录测试

1. 创建一个节点
2. 退出登录
3. 重新登录
4. 如果节点还在 → ✅ 已保存到数据库
5. 如果节点消失 → ❌ 还在 localStorage

### 方法 2: 查看网络请求

1. 按 F12 打开开发者工具
2. 切换到 Network（网络）标签
3. 创建一个节点
4. 查看是否有请求发送到 `http://localhost:8000/api/nodes`
5. 如果有 → ✅ 正在同步到数据库

### 方法 3: 检查 Token

在浏览器控制台输入：

```javascript
localStorage.getItem('fastapi_token')
```

- 如果返回一长串字符 → ✅ 已连接到 FastAPI
- 如果返回 `null` → ❌ 还在使用旧的登录方式

## 🐛 常见问题

### Q: 迁移工具打不开？

A: 确保文件在正确的位置：
```
../postgraduate/收集/smart-tree/migrate_to_backend.html
```

### Q: 迁移时提示 "未找到登录 token"？

A: 需要先重新登录：
1. 退出登录
2. 重新登录
3. 再打开迁移工具

### Q: 迁移后节点还是消失？

A: 检查：
1. 后端是否运行
2. 是否真的重新登录了
3. 浏览器控制台是否有错误

### Q: 我不想迁移，直接清空可以吗？

A: 可以！如果数据不重要：
1. F12 → Application → Local Storage
2. 删除 `smart-tree-storage`
3. 刷新页面
4. 重新创建
