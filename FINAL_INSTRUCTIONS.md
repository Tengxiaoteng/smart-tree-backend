### 步骤 2: 打开应用

### 步骤 3: 同步数据到数据库

你会在页面**右下角**看到一个蓝色的浮动按钮：

```
┌─────────────────────────┐
│  🔄 同步到数据库         │
│                         │
│  💡 提示：              │
│  • 点击按钮将本地数据    │

│  • 同步后可以在任何设备  │

│  • 退出登录后数据不会    │
│    丢失                  │
└─────────────────────────┘
```

**点击这个按钮**，等待同步完成。

### 步骤 4: 验证数据已保存

1. 同步完成后，点击右上角的用户菜单
2. 点击 "退出登录"
3. 重新登录
4. **检查你的知识树节点是否还在** ✅

如果节点还在，说明数据已成功保存到数据库！🎉

## 📊 功能说明

### 自动同步

从现在开始，每次你创建、更新、删除节点时，都会**自动同步到数据库**。

你可以在浏览器控制台（F12）看到同步日志：

```
节点已同步到服务器: 机器学习基础
```

### 手动同步

如果你想确保所有数据都已同步，可以点击右下角的 "🔄 同步到数据库" 按钮。

这会将所有本地数据批量同步到数据库。

### 查看同步状态

打开浏览器控制台（F12），你会看到详细的同步日志：

```
开始同步 2 个主题和 15 个节点到服务器...
✓ 主题已同步: 机器学习
✓ 根节点已同步: 机器学习
✓ 子节点已同步: 机器学习基础概念
✓ 子节点已同步: 监督学习详解
...
✅ 所有数据已同步到服务器！
```

## 🔍 故障排查

### 同步按钮不显示？

- 确保前端已重启
- 刷新浏览器页面（Ctrl+R 或 Cmd+R）

### 同步失败？

1. 检查后端是否运行：
   ```bash
   curl http://localhost:8000/health
   ```

2. 检查是否已登录：
   ```javascript
   // 在浏览器控制台执行
   localStorage.getItem('fastapi_token')
   ```

3. 查看控制台错误信息（F12 → Console）

### 节点还是消失？

1. 确认已点击同步按钮
2. 确认同步成功（显示绿色提示）
3. 查看控制台是否有错误
4. 检查 Network 标签，确认请求发送到后端
